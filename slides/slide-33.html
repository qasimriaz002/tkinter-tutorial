Lambda Functions in GUI

The command property of a Button expects a **reference to a function**, not the result of a function call.

The Common Mistake

If you write command=my_function("Hello"), Python executes my_function immediately when the script loads to determine what value to assign to command. The button then does nothing when clicked.

The Solution: Lambda

A lambda creates an anonymous wrapper function. When you write command=lambda: my_function("Hello"), you are telling Tkinter: "Store this small function, and run the code inside it only when the button is clicked."

Rule of Thumb: Use command=callback (no parentheses) if the function takes no arguments. Use command=lambda: callback(args) if the function requires arguments.

import tkinter as tk # --- THE FUNCTION --- def show_message(text): print(text) root = tk.Tk() # --- 1. INCORRECT WAY (Immediate Execution) --- # This prints "Hello" to the console IMMEDIATELY when script runs. # The button receives "None" as the command, so clicking does nothing. btn_bad = tk.Button( root, text="Bad Button", command=show_message("Hello") # <-- Runs now! ) btn_bad.pack(pady=5) # --- 2. CORRECT WAY (Using Lambda) --- # The lambda creates a temporary function that wraps the call. # It only runs "show_message" when the button is actually clicked. btn_good = tk.Button( root, text="Good Button", command=lambda: show_message("Hello") # <-- Waits for click! ) btn_good.pack(pady=5) root.mainloop()
