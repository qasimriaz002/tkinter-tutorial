Databases & SQL Concepts

What & Why?

A Database is an organized collection of data. Unlike raw text files, databases enforce structure, security, and speed.

  • File Handling Issues: Redundancy (duplicate data), Corruption (simultaneous editing), Slow Search (scanning whole text).
  • Database Solutions: Atomicity (all-or-nothing transactions), Indexing (instant search), Relationships.

SQL Structure

  • DDL (Data Definition): Create/Drop tables (Structure).
  • DML (Data Manipulation): Insert/Update/Delete data.
  • DQL (Data Query): SELECT (Retrieve data).

Key Concepts

  • Primary Key (PK): Unique ID for every row (e.g., Student ID). Cannot be null.
  • Foreign Key (FK): Links one table to another (e.g., Student ID in 'Marks' table).
  • Not Null: Field must contain data.

import sqlite3 # Connect (Auto-creates .db file) conn = sqlite3.connect("school.db") c = conn.cursor() # --- 1. CREATE (DDL) --- # id is PRIMARY KEY (Unique), name is NOT NULL c.execute('''CREATE TABLE IF NOT EXISTS students (id INTEGER PRIMARY KEY, name TEXT NOT NULL, age INT)''') # --- 2. INSERT (DML) --- # Insert multiple records students = [("John", 22), ("Sarah", 21), ("Mike", 23)] c.executemany("INSERT INTO students (name, age) VALUES (?, ?)", students) # --- 3. SELECT (DQL) --- # A. Select specific columns c.execute("SELECT name FROM students") print("Names:", c.fetchall()) # B. Select with WHERE clause (Filter) c.execute("SELECT * FROM students WHERE age > 21") print("Students > 21:", c.fetchall()) # Commit and Close conn.commit() conn.close()

Code Explanation:

1. Connection & Setup

  • sqlite3.connect(): Establishes the link. If school.db doesn't exist, it creates it automatically in your project folder.
  • cursor(): The "cursor" object acts like a pen that writes to the database. All SQL commands are executed through it.

2. Creating Structure (DDL)

  • IF NOT EXISTS: Prevents the code from crashing if you run it twice; it only creates the table the first time.
  • INTEGER PRIMARY KEY: Automatically assigns a unique ID (1, 2, 3...) to every student. No two students can have the same ID.
  • NOT NULL: Ensures the name column cannot be left empty.

3. Inserting & Querying Data

  • executemany: An efficient way to insert a list of data tuples in one go instead of writing 3 separate INSERT commands.
  • VALUES (?, ?): The ? is a placeholder. This prevents "SQL Injection" attacks by treating inputs as data, not code.
  • WHERE age > 21: A filter clause that tells the database to return only rows matching this condition.
  • commit(): Critical. Changes are temporary until you commit. If you forget this, the data will vanish when the script ends.
