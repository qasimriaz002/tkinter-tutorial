SQLite Database & GUI

Building a fully functional database application in Python requires combining sqlite3 for data persistence and tkinter for the user interface.

The "Student" Table Structure

We will create a table student in university.db with the following columns:

  • reg# (Primary Key): Unique Registration Number.
  • name, dob, degree, city, phone#: Student details.
  • timestamp: Auto-generated date/time of record creation.

The Treeview Widget

The ttk.Treeview widget displays tabular data in rows and columns. It mimics a database table layout and is the standard for showing SQL query results in Tkinter.

Application Logic

  • Create: Initializes the database file and table structure.
  • Insert: Adds a new row with a timestamp.
  • Retrieve: Selects all rows and refreshes the Treeview.
  • Search: Filters data by Registration #.
  • Delete/Update: Modifies specific rows based on the Primary Key.

Code Explanation:

1. Database & Table Creation

  • reg INTEGER PRIMARY KEY: Ensures every student has a unique ID and allows fast searching.
  • create_table(): Uses IF NOT EXISTS to prevent errors if the file already exists.

2. Treeview Display

  • tree.get_children(): Returns a list of all rows currently displayed.
  • tree.delete(item): Essential loop to clear the table before loading new data; otherwise, rows will duplicate every time you click "View".

3. CRUD Operations

  • VALUES (?,?...): Using placeholders prevents SQL injection and handles data types correctly.
  • conn.commit(): Required for INSERT, UPDATE, and DELETE to actually save changes to the disk.
  • Update/Delete Logic: Both use the WHERE reg=? clause to target the specific student entered in the Entry field.
